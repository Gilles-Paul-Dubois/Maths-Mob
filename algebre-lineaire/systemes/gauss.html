Méthode de Gauss-Jordan

Gauss-Jordan method

Procédé d'élimination de Gauss-Jordan

L'idée sous-jacente est de transformer un système en un triangulaire dont la résolution est immédiate. Soit un :
α1,1x1+...+α1,nxn=β1
α2,1x1+...+α2,nxn=β2
....................................
αn,1x1+...+αn,nxn=βn
que nous pouvons encore écrire en utilisant les notations des formes linéaires:
f1(X)=β1
f2(X)=β2
...........
fn(X)=βn
Pour tout système de scalaires (k2,...,kn) ∈ Kn-1 le système est équivalent à:
f1(X)= β1
f2(X)-k2f1(X)=β2-k2β1
...............................
fn(X)-knf1(X)= βn-knβ1
Peut-on choisir les scalaires (k2,...,kn), de façon à ce que tous les coefficients de x1 soient nuls dans les équations autres que la première ?
La réponse est positive pourvu que α1,1 soit inversible dans K.
Il suffit de prendre:
k2=α2,1/α1,1
..........
kn=αn,1/α1,1
Le système initial est alors équivalent à un système du type:
α1,1x1+ α1,2x2+...+α1,nxn=β1
0 +γ2,2x2+...+γ2,nxn=δ2
....................................
0 +γn,2x2+...+γn,nxn=δn
On voit donc qu'on est ramené à résoudre le système d'ordre n-1:
γ2,2x2+...+γ2,nxn=δ2
....................................
γn,2x2+...+γn,nxn=δn
x1 sera alors calculé en fonction de x2,...,xn par:
x1=(β1-(α1,2x2+...+α1,nxn))/α1,1
Supposons maintenant qu'on ait α1,1=0. Nous affirmons que forcément un des coefficients αi,1 pour 2≤i≤n est non nul.
Dans le cas contraire toute la première colonne de la matrice serait nulle, la matrice ne serait pas de rang n et on n'aurait pas un système de Cramer, contrairement à l'hypothèse.
Cela vu, dans un système toutes les lignes sont interchangeables. Il suffit donc de commencer par échanger la ligne 1 avec la ligne i pour être placé dans les conditions α1,1≠0.
Il suffit alors d'itérer le processus, c'est à dire d'appliquer le même traitement au système:
γ2,2x2+...+γ2,nxn=δ2
....................................
γn,2x2+...+γn,nxn= δn
A chaque fois on fait tomber d'une unité l'ordre du système à résoudre, jusqu'à parvenir à:
αxn=β
dont la solution est évidente.

Gauss-Jordan elimination process

The underlying idea is to transform a system into a triangular system the resolution of which is straightforward. Consider a :
α1,1x1+...+α1,nxn=β1
α2,1x1+...+α2,nxn=β2
...................................
αn,1x1+...+αn,nxn=βn
which we can still write using the notations of linear forms:
f1(X)=β1
f2(X)=β2
...........
fn(X)=βn
For any system of scalars (k2,...,kn) ∈ Kn-1 the system is equivalent to:
f1(X)= β1
f2(X)-k2f1(X)=β2-k2β1
...............................
fn(X)-knf1(X)= βn-knβ1
Can we choose the scalars (k2,...,kn), so that all the coefficients of x1 are zero in the equations other than the first?
The answer is positive provided that α1.1 is invertible in K.
Just take:
k2=α2.1/α1.1
..........
kn=αn,1/α1,1
The initial system is then equivalent to a system of the type:
α1,1x1+ α1,2x2+...+α1,nxn=β1
0 +γ2,2x2+...+γ2,nxn=δ2
...................................
0 +γn,2x2+...+γn,nxn=δn
We therefore see that we are reduced to solving the system of order n-1:
γ2,2x2+...+γ2,nxn=δ2
...................................
γn,2x2+...+γn,nxn=δn
x1 will then be calculated as a function of x2,...,xn by:
x1=(β1-(α1,2x2+...+α1,nxn))/α1,1
Suppose now that we have α1,1=0. We affirm that necessarily one of the coefficients αi,1 for 2≤i≤n is nonzero.
Otherwise the entire first column of the matrix would be zero, the matrix would not be of rank n and we would not have a system of Cramer, contrary to the hypothesis.
This being seen, in a system all the lines are interchangeable. It is just enough to swap row 1 with row i to be placed under the conditions α1.1≠0.
We can then iterate the process, i.e. to apply the same treatment to the system:
γ2,2x2+...+γ2,nxn=δ2
...................................
γn,2x2+...+γn,nxn= δn
Each time we drop the order of the system to be solved by one unit, until we reach:
αxn=b
the solution of which is obvious.
définition 1
Cet algorithme est connu sous le nom de 'procédé d'élimination de Gauss-Jordan' ou encore de 'pivot de Gauss' , le 'pivot' correspondant au coefficient non nul dont nous avons besoin à chaque fois pour itérer le processus.
definition 1
This algorithm is known as 'Gauss-Jordan elimination process' or even of 'Gaussian pivot' , the 'pivot' corresponding to the non-zero coefficient that we need each time to iterate the process.
Mesurons maintenant l'efficacité de cet algorithme.
A la première étape nous avons n-1 divisions puis n2 multiplications puis n additions. Les termes en n sont négligeables devant n2 . Le procédé total est donc en n2+(n-1)2+...+22+12=n(n-1)(2n-1)/6. L'efficacité est donc en n3/3, ce qui est bien meilleur que le n! des formules de Cramer nécessitant le calcul de déterminants.
On peut maintenant réfléchir à de possibles optimisations de ce procédé. Par exemple, au lieu de se borner à prendre un pivot non nul, on préférera, si le cas se présente, prendre un pivot égal à 1. Cela dit, si aucun pivot n'est égal à l'unité entre tous les pivots possibles, on aura intérêt à en choisir un qui ne soit ni trop petit ni trop grand pour ne pas avoir de problèmes d'arrondis trop importants en fonction de la précision des types numériques utilisés (float, double, etc...).
Let us now measure the efficiency of this algorithm.
At the first stage we have n-1 divisions then n2 multiplications then n additions. The terms in n are negligible in front of n2 . The total process is therefore in n2+(n-1)2+...+22+12=n(n-1)(2n-1)/6. The efficiency is therefore n3/3, which is much better than n! using Cramer formulas requiring the calculation of determinants.
We can now think about possible optimizations of this process. For example, instead of just taking a nonzero pivot, it is preferable, if the case arises, to take a pivot equal to 1. This being noticed , if no pivot is equal to unity among all possible ones, it would be in your interest to choose one that is neither too small nor too big not to have too big rounding problems in depending on the precision of the numeric types used (float, double, etc...).

Note that the above implementation is 'destructive', the system is altered by the resolution process. This has no importance when one is only interested in solutions. In general the systems to be solved are (very) large, one can imagine that they are stored on disk. It is then preferable, in order not to use too much memory space to use a destructive process.
