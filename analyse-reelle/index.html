Addendum July 2021

We believe that the Julia language has a great future. This is why, for this module, all the examples given in Python have been duplicated in Julia with the current version 1.6 both for the course and for the programming exercises.

On this occasion we checked the full backward compatibility of Python version 3.4 with the current version 3.8.

It is possible to test the examples and modify them online, by simply copying and pasting them on Jupyter's site. It is therefore not necessary to install any IDE in local mode, although this can be done from their site.

To date, there are several interesting free multi-language (python/Julia) and multi-platform (Windows, Linux, Mac) editors, including Pycharm by Jetbrains And atom . Just install the corresponding plugins (juno for atom, julia plugin for jetbrains).

Under Linux, the program notepadqq (Linux version of notepa++) can edit Julia programs. Moreover, with suitable settings, these programs can be run without leaving the editor. notepadqq can also be used in conjunction with PyCharm editing one by one the files of a Pycharm Julia project.

Anaconda also provides a recent and complete Python environment and gives access to Jupyter if the IJulia package is installed, as well as Pycharm.

The reader will verify that the single 'MTH229' package from Julia advantageously replaces the numpy, scipy, random, functools libraries of Python (and perhaps others). Many primitives of these libraries are already deprecated before their announced removal, which will cause update problems.

Do not hesitate to inform us of any erratic operation of the programs given as examples. Thanks in advance!
