Définitions

La mémoire des ordinateurs se présente comme une suite 'd'octets' (anglais 'bytes').
définition
Un 'bit' (contraction de 'binary digit') est un chiffre binaire c'est à dire 0 ou 1.
définition
Un 'octet' est une suite de 8 'bits'.
Un octet se présente donc, par exemple, sous la forme:
0 1 1 0 1 0 0 1
C'est donc un nombre binaire à 8 chiffres maximum.
La plus petite valeur possible est 0 et la plus grande 11111111=255, soit au total 28 =256 possibilités.
C'est évidemment trop peu pour représenter les entiers naturels.
La plupart des langages de programmation (typés) proposent des types prédéfinis pour représenter les entiers naturels (ou du moins une partie d'entre eux).
Le langage C, par exemple, propose les unsigned short int (généralement sur 2 octets) et les unsigned long int (sur 4 octets).
Disons qu'un entier court peut être vu comme une suite de 16 digits ou comme un nombre à deux chiffres en base 256 au choix, et qu'un entier long peut être vu comme un nombre binaire à 32 chiffres .
Le nombre total des possibilités pour les entiers courts est 216=65536, et le plus grand entier naturel représentable est 65535.
Le nombre total des possibilités pour les entiers longs est: 232=4294967296, et le plus grand nombre représentable est 4294967295.
Il peut donc y avoir des débordements, certains langages, pour ne citer que C, ne traitent pas ces situations, c'est à dire qu'ils proposent des résultats incohérents sans message d'erreur, il appartient donc au programmeur de prévoir et de gérer lui-même les cas particuliers qui peuvent se produire.
Signalons encore deux particularités.
La première c'est que les entiers sont écrits en mémoire à l'envers les octets sont écrits par ordre de poids croissants (un peu comme un nombre qui serait écrit de droite à gauche.
La seconde c'est que parallèlement au système binaire on utilise beaucoup le système hexadécimal (base 16) qui est une puissance de deux, ainsi chaque 'chiffre' hexa est composé de 4 chiffres binaires, et un octet se note avec seulement deux chiffres hexas.
En hexa on utilise les chiffres décimaux usuels 0,1,2,.....,9 puis on complète avec des lettres:
A=10,B=11, ..... F=15
Ainsi le plus grand octet est FF=255

Definitions

Computer memory can be seen as as a series of 'bytes'.
definition
One 'bit' is a binary digit, i.e. 0 or 1.
definition
One 'byte' is a sequence of 8 'bits'.
A byte therefore takes the form, for example:
0 1 1 0 1 0 0 1
It is therefore a binary number with a maximum of 8 digits.
The smallest possible value is 0 and the largest 11111111=255, i.e. a total of 28 =256 possibilities.
This is obviously not enough to represent the natural numbers.
Most of the (typed) programming languages ​​provide predefined data-types to represent natural integers (or at least a part of them).
The C language, for example, offers the unsigned short int (usually on 2 bytes) and the unsigned long int (on 4 bytes).
Let's say that a short integer can be seen as a sequence of 16 digits or as a two-digit number in base 256 as you prefer and a long integer can be seen as a 32-digit binary number.
The total number of possibilities for short integers is 216 =65536, and the largest representable natural number is 65535.
The total number of possibilities for long integers is: 232=4294967296, and the largest representable number is 4294967295.
Therefore we can have overflows, certain languages, to name a few than C, do not deal with these situations, i.e. they propose inconsistent results with no error message, so it is up to the programmer to foresee and manage himself the particular cases which can occur.
Let us point out two more particularities.
The first is that the integers are written in memory backwards bytes are written in ascending order of weight (much like a number that would be written from right to left.
The second is that parallel to the binary system we use a lot the system hexadecimal (base 16) which is a power of two, so each 'digit' hex is made up of 4 binary digits, and a byte is noted with only two hex digits.
In hexa we use the usual decimal digits 0,1,2,....,9 then we complete with letters:
A=10,B=11, ..... F=15
So the largest byte is FF=255

Le coin de Python

Le langage Python présente une particularité (parmi d'autres). Il possède un type entier (int) prédéfini comme la plupart des autres langages, mais il gère les débordements de façon très originale. Dès que la capacité maximale est atteinte, il change de mode de représentation et passe en 'mode liste' dans lequel on n'est limité que par la taille mémoire de la machine.

Python's corner

The Python language has a particularity (among others). It has a predefined integer (int) type like most others languages, but it handles overflows in a very original way. As soon as the maximum capacity is reached, it changes mode of representation and switches to 'list mode' in which one is only limited by the memory size of the machine.
