Nous avons vu dans les pages qui précèdent que les notions de fonction et d'application sont parfaitement synonymes.
Cependant, nous ne pouvons passer sous silence certaines pratiques, certains usages qu'on rencontre surtout chez les auteurs d'ouvrages scolaires de l'enseignement secondaire.
Ils définissent des fonctions, le plus souvent numériques de ℝ dans ℝ , au moyen d'une formule n'ayant de sens que pour certaines valeurs de la variable , comme par exemple:
f(x) = √(1+tan(x)).
Ils posent alors des questions du genre "Quel est le domaine de définition de f ?".
Ceci n'a pas grand sens du point de vue de la théorie générale des relations puisqu'une application (fonction) est définie a priori par son ensemble source, son ensemble but, et une relation fonctionnelle.
La question devrait donc être formulée ainsi: "Pour quelles valeurs de x la formule a-t-elle un sens ?". Puis, "en déduire une application de cet ensemble vers ℝ définie par... etc.."
Mais pour faire court, la question sera souvent posée comme il est dit plus haut.
Ajoutons enfin une précision concernant la définition des fonctions au moyen de formules.
Nous pouvons parler, par exemple, de la fonction x → sin(x). Remarquons ici que la lettre x joue le rôle d'une 'variable muette' pouvant être remplacée par n'importe quelle autre.
Ainsi x → sin(x) et y → sin(y) désignent une seule et même fonction, celle qui à tout élément associe son sinus. Le fait d'utiliser une variable permet d'utiliser une formule symbolique, et d'éviter la lourdeur du langage naturel dès que la formule devient un peu compliquée.
Par ailleurs il ne faut pas croire que toute fonction peut être définie par une formule.
Considérons par exemple la fonction dite 'de Dirac' ainsi définie:
δ (x) = 1 si x ∈ ℚ et 0 si x ∉ ℚ
Il est illusoire d'essayer d'écrire une telle fonction par une formule, aussi bien que de la représenter graphiquement d'ailleurs.
Certains vont même jusqu'à dire qu'une fonction est une application "non partout définie", c'est à dire que tout élément de la source possède au plus une image.
De toutes façons la notion d'application est la seule qui soit vraiment importante. De fait, toute relation fonctionnelle peut être définie par une application .

We have seen in the preceding pages that the notions of function and application are perfectly synonymous.
However, we cannot ignore certain practices, certain uses that one encounters especially among the authors of textbooks for secondary schools.
They define functions, most often numerical, of ℝ in ℝ , using a formula having no meaning only for certain values ​​of the variable , like
f(x) = √(1+tan(x)).
They then ask questions like "What is the domain of f?
This does not make much sense from the point of view of the general theory of relations since an application (function) is defined a priori by its departure set (domain) , its destination set (codomain), and a functional relationship.
The question should therefore be formulated as follows: "For which values ​​of x does the formula make sense?". Then, "deduce an application of this set to ℝ defined by... etc.."
But in short, the question will often be asked as it is said earlier.
Finally, let's add a precision concerning the definition of functions by mean of formulas.
We can speak, for example, of the function x → sin(x). Note here that the letter x plays the role of a 'dummy variable' can be replaced by any other.
Thus x → sin(x) and y → sin(y) denote one and the same function, that which associates with any element its sine. Using a variable allows you to use a symbolic formula, and to avoid the heaviness of natural language as soon as the formula gets a little complicated.
Moreover, it should not be believed that any function can be defined by a formula.
Consider for example the so-called 'Dirac' function thus defined:
δ (x) = 1 if x ∈ ℚ and 0 if x ∉ ℚ
It is illusory to try to write such a function by a formula, as well as representing it graphically.
Some even go so far as to say that a function is an application "not everywhere defined", i.e. any element of the source has at most one image.
In any case, the notion of application is the only one that is really important. Fact, all functional relationship can be defined by an application .
